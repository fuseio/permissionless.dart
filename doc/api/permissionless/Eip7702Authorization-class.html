Eip7702Authorization class

EIP-7702 Authorization for EOA code delegation.

An authorization allows an EOA to delegate its code to a smart contract implementation. When included in a transaction, the EOA temporarily "becomes" the smart contract for the duration of the transaction.

This enables EOAs to use smart account features without deploying a separate contract or changing their address.

Example:

final auth = await Eip7702Authorization.sign(
  chainId: BigInt.from(1),
  contractAddress: Simple7702AccountAddresses.defaultLogic,
  nonce: BigInt.zero,
  privateKey: ownerPrivateKey,
);

// Include auth in transaction to enable smart account features
